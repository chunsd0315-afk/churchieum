# Churchieum / 교회이음

**교회를 잇고, 말씀을 잇고, 믿음을 잇다.**

> **Current version:** Churchieum 1.0 Beta

---

## 소개

**교회이음(Churchieum)**은 단순한 교회 관리 프로그램이 아니라, 교회와 가정, 목회자와 성도, 성도와 성도, 현재 세대와 다음 세대를 잇는 **신앙 공동체 플랫폼**입니다.

행정 도구를 넘어, 말씀·기도·교제·섬김이 일상 속에서 이어지도록 돕는 것이 교회이음의 목표입니다.

---

## 비전 (Project Vision)

교회이음은 다음을 꿈꿉니다.

- **교회와 가정이 하나로** — 예배당 안팎의 신앙 생활이 끊기지 않습니다.
- **목회자와 성도가 함께** — 목회와 양육이 기술이 아니라 관계 위에서 이루어집니다.
- **성도와 성도가 서로를 세움** — 공동체 안에서 기도와 나눔이 흐릅니다.
- **세대를 잇는 믿음** — 현재 세대의 은혜가 다음 세대로 전해집니다.

---

## 주요 목적 (Main Purpose)

| 목적 | 설명 |
|------|------|
| **말씀** | 성경 읽기, 성경 통독, 설교, 은혜 기록을 통해 하나님의 말씀이 삶이 되도록 합니다. |
| **기도** | 개인·교회·중보 기도제목을 나누고 함께 기도합니다. |
| **교제** | 공지, 주보, 일정, 앨범, 교회 나눔으로 공동체의 소식과 관계를 이어갑니다. |
| **섬김** | 조직·성도·교역자 관리를 통해 목회와 섬김이 질서 있게 이루어지도록 돕습니다. |
| **양육** | 은혜 기록, QT, 성경 통독 등으로 신앙 성장을 기록하고 돌아봅니다. |

## 사용자 역할 (User Roles)

교회이음은 세 가지 역할을 기준으로 경험을 나눕니다.

### 관리자 (Admin) — `super_admin`

- 교회 전체 설정, 조직 구조, 교역자·성도·초대 관리
- 통계·보고서, 교회 인증 등 운영 기능
- 콘텐츠(공지, 주보, 일정, 설교 등) 전체 관리
- 성도 모드와 관리자 모드 전환 가능

**체험 계정:** `pastor01@churchieum.com` / `Church@2026`

### 교역자 (Pastor) — `pastor`

- 담당 교구·구역·부서 범위의 목회·양육
- 공지·일정·앨범 등 범위 내 콘텐츠 작성 (권한 정책에 따름)
- 성도의 은혜 기록·성경 통독 등 양육 관련 기능 접근
- 교회 나눔 게시물 작성·관리

**체험 계정:** `pastor02@churchieum.com` / `Church@2026`

### 성도 (Member) — `member`

- 설교, 은혜 기록, 기도, 공지, 주보, 일정, 성경, 앨범 등 신앙 생활 기능
- 프로필·소속 정보, 교회 정보 확인
- 교회 나눔 참여

**체험 계정:** `member60@demo.com` / `Church@2026`

## 주요 기능 (Main Features)

### 성도·교역자 모드

- **홈** — 맞춤 대시보드, 최근 공지·일정·기도 미리보기
- **설교** — 예배 설교 말씀 열람
- **은혜 기록** — 말씀과 삶 속 은혜 기록·나눔
- **기도** — 내 기도, 교회 기도, 중보 기도
- **공지사항** — 교회 소식·안내
- **주보** — 예배 순서·주간 소식
- **일정** — 예배·행사 일정
- **성경** — 성경 읽기·검색·저장
- **성경 통독** — 통독 계획·진행률
- **앨범** — 교회 공동체 사진
- **교회 나눔** — 교회 간·공동체 간 나눔
- **내 정보 / 교회 정보** — 프로필·소속·교회 기본 정보

### 관리자 모드

- **조직 관리** — 교구, 구역, 부서
- **교역자·성도·초대 관리**
- **설교·공지·주보·일정·기도·앨범·성경 통독** 콘텐츠 관리
- **통계·보고서**, **교회 인증**
- **교회 설정** — 조직 명칭, 환경 설정

## 개발 철학 (Development Philosophy)

1. **관계 우선** — 기능보다 공동체의 연결을 먼저 생각합니다.
2. **말씀 중심** — 모든 화면과 흐름이 말씀·기도·은혜를 돕는 방향으로 설계됩니다.
3. **단순하고 따뜻한 UX** — 복잡한 행정 도구가 아니라, 매일 열어도 부담 없는 경험을 지향합니다.
4. **점진적 성장** — Beta 단계에서는 데모·로컬 데이터와 백엔드(Supabase)를 병행하며, 안정적으로 기능을 확장합니다.
5. **교회 현장의 언어** — 개발·문서·UI 모두 교회 현장에서 쓰는 말을 존중합니다.

## 접근성 원칙 (Accessibility)

교회이음은 **연세 많은 목회자와 성도**도 편히 쓸 수 있도록 다음을 지향합니다.

- **큰 글씨·넉넉한 터치 영역** — 버튼과 메뉴를 누르기 쉽게
- **짧고 분명한 문장** — 한 화면에 한 가지 목적
- **익숙한 교회 용어** — 행정·IT 전문 용어 대신 일상적인 표현
- **모바일·PC 모두 지원** — 스마트폰이 익숙하지 않은 분도 PC에서 사용 가능
- **시각적 계층 단순화** — 중요한 정보(공지, 주보, 기도)를 먼저 보여 줌

## 로컬 개발

### 요구 사항

- Node.js 18+
- npm

### 설치 및 실행

```bash
npm install
npm run dev
```

브라우저에서 http://localhost:5173 을 엽니다.

### Supabase (선택)

백엔드 연동 시 프로젝트 루트에 `.env` 파일을 만들고 다음을 설정합니다.

```env
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

환경 변수가 없어도 앱은 실행되며, 데모·localStorage 데이터로 주요 화면을 체험할 수 있습니다.

### 기타 스크립트

```bash
npm run build      # 프로덕션 빌드
npm run preview    # 빌드 결과 미리보기
npm run typecheck  # TypeScript 검사
npm run lint       # ESLint
```
